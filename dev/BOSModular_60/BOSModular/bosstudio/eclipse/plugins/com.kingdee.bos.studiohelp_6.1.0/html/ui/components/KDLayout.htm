    KDLayout

父类

布局约束类：

   KDLayout.Constraints

 

 

 

功能列表

   通过为每一个控件添加一个停靠约束（Anchor），把设计时的绝对定位布局转化为可缩放的相对布局。

 

使用指南

  KDLayout是专门为UIDesigner设计布局管理器，通过对设计面板上的每一控件添加一个anchor属性，KDLayout就能将UIDesigner上绝对定位的布局变成可缩放的布局。如果在UIDesigner外单独使用KDLayout，你需要填写更多的属性。

 

KDLayout布局属性

       KDLayout的内部类Constraints定义了在使用KDLayout时容器上每个控件必须填写的布局属性。

 

OriginalBound

       表示的是控件在UIDesigner上的原始大小

Anchor

       表示的是控件的锚定方式。

       锚定是指控件上的一边与父容器的同方向的一边或是中心点之间的固定关系。当控件的锚定后，控件位置相对父容器固定。当父容器大小改变后，控件与父容器之间的固定关系不变，控件的位置大小由父容器所决定。

在KDLayout中定义了五个锚定点、两种锚定方式，共九个锚定方式可选择。不同锚定的方式用不同的常量表示，而且可以通过逻辑运算符“或”进行搭配使用。

    ANCHOR_TOP ：

控件的上边与父容器的上边距离固定，等同于设计时距离

    ANCHOR_BOTTOM ：

控件的下边与父容器的下边距离固定，等同于设计时距离

    ANCHOR_LEFT ：

控件的左边与父容器的左边距离固定，等同于设计时距离

ANCHOR_RIGHT ：

控件的右边与父容器的右边距离固定，等同于设计时距离

ANCHOR_CENTRE ：

控件的中心点与父容器的中心点水平距离固定，等同于设计时距离

    ANCHOR_TOP_SCALE :

控件的上边与父容器的上边距离固定，等同于设计时距离乘【父容器高度与设计时高度的缩放比】。

    ANCHOR_BOTTOM_SCALE :

控件的下边与父容器的下边距离固定，等同于设计时距离乘【父容器高度与设计时高度的缩放比】。

    ANCHOR_LEFT_SCALE :

控件的左边与父容器的左边距离固定，等同于设计时距离乘【父容器宽度与设计时宽度的缩放比】。

    ANCHOR_RIGHT_SCALE :

控件的右边与父容器的右边距离固定，等同于设计时距离乘【父容器宽度与设计时宽度的缩放比】。

 

  除了每个控件的布局信息，控件的布局容器也需要填写一个ClientProperty：“OriginalBounds”用于记录UIDesigner设计时父容器（设计面板）的大小。

      

API说明

参见产生的通用控件JavaDoc文档。

 

示例

       三个键钮，一个左对齐、一个右对齐、一个居中

       cons = new KDLayout.Constraints();

    cons.originalBounds = new Rectangle(19, 27, 71, 25);

    cons.anchor = KDLayout.Constraints.ANCHOR_LEFT;

    b11.putClientProperty("KDLayoutConstraints", cons);

   

 

    cons = new KDLayout.Constraints();

    cons.originalBounds = new Rectangle(162, 27, 71, 25);

    cons.anchor = KDLayout.Constraints.ANCHOR_CENTRE_SCALE;  

    b12.putClientProperty("KDLayoutConstraints", cons);

   

 

    cons = new KDLayout.Constraints();

    cons.originalBounds = new Rectangle(314, 26, 71, 25);

    cons.anchor = KDLayout.Constraints.ANCHOR_RIGHT;  

b13.putClientProperty("KDLayoutConstraints", cons);

 

 

    getContentPane().add(b13, null);

    getContentPane().add(b12, null);

    getContentPane().add(b11, null);

           

// 填写父容器设计时的大小

((JComponent)this.getContentPane()).putClientProperty("OriginalBounds", new Rectangle(0, 0, 397, 297));

getContentPane().setLayout(new KDLayout());

 

// 设置容器的最佳大小，当容器大小小于最佳大小时出现滚动条

((JComponent)getContentPane()).setPreferredSize(new Dimension(397, 297));

setContentPane(new JScrollPane(getContentPane())); 

 

 

虽然在上面的例子中控件的布局约束是通过ClientProperty的方式添加的；但是KDLayout也支持按传统的通过JComponent.add(Component,Object)的方式添加控件约束。
