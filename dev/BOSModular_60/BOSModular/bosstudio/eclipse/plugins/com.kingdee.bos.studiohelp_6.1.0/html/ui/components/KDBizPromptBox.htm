使用指南

       F7业务控件是在F7通用控件基础上针对业务需求实现的一个控件。它除了拥有通用控件的特性外，还支持业务数据的绑定。

       业务数据绑定相关的属性的使用：

1）  displayFormat：显示期格式，String类型，以"$fieldname$"表示填入字段，"$$"表示"$"符.取值示例："$id$+$name$"。在不可编辑时，焦点和非焦点状态的数据展现都使用displayFormat，忽略eidtFormat。

2）  editFormat：编辑期格式，其它同displayFormat。

3）  commitFormat：数据提交格式属性。它是展现格式（上面的displayFormat或editFormat都属于展现格式之一）的序列，用“;”隔开。格式为：<展现格式1>[;<展现格式2>[...]]。如：”$id$;$name$“ ，又如：”$id$+$name$;$id$+$age$”。以后者为例，在提交数据时生成过滤条件为：(id=001 and name="username") or (id=001 and age=35)。commitFormat暂不支持非String类型字段。如果设置了非String类型的字段作为匹配字段，将查询不到结果。

4）  queryInfo：查询对象的PK值，String类型。注意：F7用到的queryInfo中的主对象必须有主键字段。

5）  entityViewInfo：查询条件，EntityViewInfo类型。

6）  多选属性(enabledMultiSelection), true为支持多选。多选时data为数组, 默认展现数组第一项(下标为0)。多选到单选的切换会将数据[0]作为Data属性，其它数据丢弃。Data数据类型保持单个元素的类型。

7）  defaultF7UIName属性：设置F7弹出框的类名, 如果Selector为null, 那么将使用该类名进行反射创建一个弹出框。如果该类名为null, 那么将弹出KDCommonPromptDialog。

8）  userData属性：客户程序用于存储业务数据的一个属性，比如存储HashMap。

 

如果客户程序需要扩展功能，以下属性可以帮助你很容易的自定义控件的行为。下面是控件结构示意图：

1）  displayFormatter：显示期格式器，将控件中的data对象格式化为显示期展现在编辑框中的字串。

2）  editFormatter：编辑期格式器，将控件中的data对象格式化为编辑期展现在编辑框中的字串。

3）  commitParser：对象解析器，将编辑框中的字符串解析为保存在控件中的数据对象。默认解析器是DefaultParser，它的流程如上图所示。

 

事件：

1．选择框“将弹出”事件，发生在选择框弹出之前：

public void addSelectorListener(SelectorListener listener);

在弹出选择框之前，监听器可以修改控件的某些属性，特别是和选择框有关的属性。默认情况下的弹出框是KDCommonPromptDialog，在本事件中就可以对弹出框的queryInfo，entityViewInfo属性进行设置。监听器还可以在这时使用KDPromptBox.setSelector()设置一个新的弹出框，甚至可以不让控件弹出选择框，使用SelectorEvent.setCanceled(true)即可。

 

2. Commit事件，发生在用户在编辑框中按Enter键后：

public void addCommitListener(CommitListener listener);

在解析编辑框中的字符串之前发出该事件，事件中带有编辑框中的文本，使用CommitEvent.getText()方法获得。客户程序可以在事件处理中设置Parser及其属性。也可以通过CommitEvent.setCanceled(true)取消该次解析的执行。

典型案例分析 - 如何使用自定义的对话框

     1. 确定弹出框的继承关系

自定义弹出框如果用到了Query对象，并和KDCommonPromptDialog应用场景相似， 可以直接从KDCommonPromptDialog继承，并遵从其规范（属性设置和返回值结构）。如果完全未使用到Query对象甚至和业务对象无关，则需要从KDPromptSelector继承。

    

     2. 定制控件编辑框中的数据展现

如果弹出框从KDCommonPromptDialog继承，返回值是一个数组。单选时只有第0项有效。如果弹出框从KDPromptSelector继承，返回值应当是直接所需的data属性值。返回值通过默认格式化器按照显示期和编辑期格式转化为字符串显示在控件的编辑框中。只有当默认格式化器不满足客户程序需要时才需要自己编写IFormatter实例，并且使用KDBizPromptBox.setXxxFormatter()方法设置到控件中。实现IFormatter接口时，其中方法valueToString(Object o)的参数o就是KDPromptSelector.getData()返回的对象。

    

     3. 如何提交数据

这里提交的意思是将编辑框中的字符串变为对象存到控件data属性中。

数据提交通过IParser实现，见控件的commitParser属性。

同样KDBizPromptBox提供了默认的数据提交的实现：DefaultParser。该类的功能是将编辑框中的文本按commitFormat进行解析生成条件，然后通过查询数据库得到结果集，并构建业务对象设置到控件中。如果该解析器不满足需要则需要客户程序定制数据提交的解析器。

 

4．注意事项

自定义了格式化器和解析器后，控件属性displayFormat、editFormat、commitFormat将转变为自定义的格式化器或解析器的pattern（applyPattern方法的参数）。

 

示例

       1．直接使用业务控件：

KDBizPromptBox pb = new KDBizPromptBox();

pb.setQueryInfo("com.kingdee.bos.ctrl.extendcontrols.User");

pb.setEditFormat("$id$");

pb.setDisplayFormat("$id$+$name$");

pb.setCommitFormat("$id$+$name$;$id$+$age$");

EASv4.1 新增内容：

一 、F7支持业务对象的维护

1.    需求：从F7选择对话框可以调出业务对象的序时簿，从而进行各种新增维护删除操作，关闭序时簿返回F7对话框后自动刷新以体现最新的变化。

2.    设计实现：

A）  Query对象上添加isF7Maintain布尔型扩展属性，指示该Query是否支持业务对象维护。

B）  Query对象或者Query对象的主实体对象上添加扩展属性 “listUI”，指明该Query对象相关的业务对象需要使用哪个序时簿进行维护操作。如图：
 
 

 

 

 

 

 

 

 

 

 

 

 

 

 


  

 

 

 

 

 

 

 

C）  F7选择对话框中添加“维护” 按钮，Action 事件响应中首先读取Query对象上isF7Maintain扩展属性,如果为true，继续读取“listUI”属性， 如果Query对象上没有这个扩展属性，则从Query对象的主实体对象上去获取。注：如果isF7Maintain为false，或者Query对象和Entity上都没有 ListUI 扩展属性的值，维护按钮将不可见。

 

    

 

   

二 、 F7支持通用过滤

1. 术语解释：

1.1 原始条件：用户在点开F7之前，程序设置的过滤条件。

1.2 自定义条件：用户在F7选择对话框中弹出用通用过滤界面，在通用过滤界面上定义的过滤条件。

 

2.       需求：

2.1   F7选择对话框弹出时，根据原始条件取数。

2.2   用户在F7选择对话框中可以调用通过过滤对话框设置自定义过滤条件，返回后F7根据原始条件和自定义过滤条件的交集取数。

2.3   当没有自定义过滤条件时，按“刷新”按钮按原始条件取数，用户设置过自定义过滤条件后根据原始条件和自定义过滤条件的交集取数。

2.4   用户自定义过滤条件方案不保存，但一次调用的过程中可以缓存。

   

三、本版F7修复的bug

 

1 一个F7控件来回切换邦定不同的Query时，引发QueryInfo为空，抛异常。

     原因：程序员切换前，会 setSelector（null）， 这样下次邦定QueryInfo时 会在CreateSelector()中新实例化一个Selector，并且刷新标志没有置为true，这样在ConfigSelector（）中不会为 Selector set QueryInfo。造成 进入 Selector中 fillTable时 报出 nullPointException。

     解决办法： 在CreateSelector（）中新实例化Selector时把refresh标志置为true。

3.       F7显示日期字段时把日期、时间一起显示

在 fomatSingleObject() 中 对 字段类型进行判断，如果是 java.util.Date 及其子类，截取日期部分，显示。
