★     KDTree的模与节点

KDTree从JTree继承，它的模式及结构也是与JTree保持一致，而JTree使用的模DefaultTreeModel和节点DefaultMultableTreeNode不能满足KDTree的需求，于是扩展了KingdeeTreeModel和DefaultKingTreeNode。

 

       JTree   － DefaultTreeModel  － DefaultMutableTreeNode

       KDTree － KingdeeTreeModel － DefaultKingdeeTreeNode

 

使用KDTree，必须使用KingdeeTreeModel和DefaultKingdeeTreeNode。

FAQ：（问题征集中，欢迎提出使用中的任何问题。）

1、 KDTree中的一些节点，我想让它成为文件夹，可是当它没有子节点时，它会显示叶子节点的图标。是否有办法指定它是文件夹图标？

答：可以。我们首先想的办法可能是用DefaultKingdeeTreeNode的setCustomIcon方法来设节点的自定义图标。但是这个图标不能根据文件夹是否打开的状态而变化。实际上JTree已经提供了解决方案。

首先，treeModel中有一个asksAllowsChildren的属性，默认是false，我们要把它设为true：

((KingdeeTreeModel)tree.getModel()).setAsksAllowsChildren(true);

然后，对于希望成为文件夹的节点node1:

node1.setAllowsChildren(true); //其实默认已经就是true

这样，不管node1有没有子节点，都会以文件夹图标出现。且双击它使其打开/关闭时，图标也会变化。

 

2、 当只有一个根节点并且根节点被隐藏时，不能往根节点添加子节点。而根节点没有隐藏，或者根节点隐藏了，但它至少有一个子节点时，添加子节点的操作一切都正常。是什么原因？

答：并不是“不能添加子节点”，只是此时根节点没有展开，看不到结果而已。只要把根节点展开就行了：

tree.expandAllNodes(true, (DefaultKingdeeTreeNode)tree.getModel().getRoot());

 

3、 删除节点时，界面上为什么不刷新？而且我用了tree.validate(); tree.repaint();也无效。

答：这种情况很可能是这样使用造成的：

parentNode.remove(childNode);

因为从节点的角度去移除节点，树的模并不知道，所以会有“删不了”的情况。要改成这样：

tree.removeNodeFromParent(childNode);

顺便再说一下，KDTree提供了增删节点的几个方法，都是从模去操作的。这几个方法如下：

//删除节点:

public void removeNodeFromParent(MutableTreeNode node)

//删除节点的所有子节点:

public void removeAllChildrenFromParent(MutableTreeNode parent)

//插入节点:

public void insertNodeInto(MutableTreeNode newChild,MutableTreeNode parent,int index)

//在父节点末端插入节点:

public void addNodeInto(MutableTreeNode newChild,MutableTreeNode parent)
