功能列表

基本

上下箭头为一列和编辑框组成左右布局

上下箭头按钮宽度不变，高度随着编辑框高度变化。在窗体过小的情况下，允许按钮本身显示不全

控件中的编辑框和按钮显示效果和基本控件一致

　

Tab键切换

切换控件(非控件中的子元素)，焦点只属于整个控件，控件中的元素没有焦点状态

键盘方向键操作

焦点状态下，支持键盘方向键操作，递增/减编辑框中的数值；按下不放将持续增减。

Enter操作

焦点状态下， 按回车键确定数值，且默认下一个控件获得焦点

编辑器可设置

允许定制不同的编辑器，递增/减方法由编辑器提供

步长可设

适用于NumberSpinnerModel，其他model需要定制接口

阈值处理

当到达最大值或最小值，再次改变数值导致超出取值范围时，自动变为最大或最小值, 该功能由相应的model完成

样式

参见人机工程设计的外观

使用指南

上下选框能够是一个能通过上下箭头进行数据递增/减选择的控件，除了具备JSpinner的功能外，它增强了下面的功能：

 

1．步进设置

该功能的适用于spinner的editor的model从SpinnerNumberModel继承或实现了Stepable接口的情况。前者参见jdk1.4；后者通常是客户程序设置自定义的editor，该editor拥有一个实现Stepable接口的model。

 

2．嵌入外部控件

出于特殊需要，客户程序可能需要嵌入外部控件（对spinner配置新的编辑器或者model），该外部控件可能是一个已有的控件，也可能是editor和model分离尚未组合的程序。为了能够被spinner使用，该外部控件实现时需要注意：

・该控件model实现SpinnerModel接口。

・如果编辑器需要响应enter键并做焦点切换，自定editor需要自己实现该功能，因为控件不能“越权”定义客户编辑器的行为。

・如果editor本身没有和model关联，在构造editor时需要添加数据改变监听器以响应数据变化来改变显示文本。

・如果需要支持可调步长，model需要实现Stepable接口。

・如果插入到spinner中的editor是组合控件，建议覆盖hasFocus方法，以使控件正确返回focus状态。
