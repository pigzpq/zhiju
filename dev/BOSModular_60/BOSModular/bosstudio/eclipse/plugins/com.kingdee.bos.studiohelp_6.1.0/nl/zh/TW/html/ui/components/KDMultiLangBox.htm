使用指南

       多語言輸入框提供了存儲多對「特定語言-該語言相關數據」映射的功能，適用於需要多語言支持的產品。

多語言輸入框根據語言對像 LanguageInfo來標識不同的語言。LanguageInfo包含語言區域和顯示圖標。區域是唯一標識一種語言的關鍵字；圖標可以自定義，使用setIcon方法定製圖標。默認使用Locale.getDisplayName(getDefaultLocale())的前兩個字母作圖標。

和JComboBox的Item相對應，多語言輸入框使用MultiLangItem，它包含語言對像(LanguageInfo)和對應數據(Object)。

 

多語言輸入框有以下功能來支持客戶程序：

·添加一種語言 addLanguage，在列表中查找，如果沒有則添加，有則不變。

·添加語言和相關數據 addItem，在列表中查找，如果沒有則添加，並放入數據，有則只放入數據。

·刪除一種語言及其關聯數據 removeLanguage,removeItem，列表中找不到被刪除項，則忽略；否則刪除。

·設置當前語言 setSelectedLanguage，指定在編輯框中顯示的語言。

·設置當前內容 setSelectedItemData，修改在編輯框中顯示的語言的數據。

 

為了方便使用，多語言控件被包裝為KDBizMultiLangBox和KDBizMultiLangArea兩個實用控件，前者是單行，後者是多行。也可以使用KDBizMultiLangBox(KDMultiLangBox.MULTILINE)來構造一個多行多語言控件，等同於KDBizMultiLangArea。使用時，應用框架首先初始化語言列表，使用createLanguageList方法。該方法有兩種形式，帶圖標參數的形式允許客戶程序指定每個區域對應的語言圖標，如下：

public static List createLanguageList(Locale[] loc, Icon[] icon);

需要注意的是：

1. 初始化後語言列表不可更改。

2. 初始化前如果未對語言列表進行創建，那麼new出來的控件就沒有任何語言。

3. 如果應用框架初始化的語言列表不夠用，控件可以通過addLanguage方法添加語言。

4. 如果應用框架有約定整個應用使用統一的語言列表，那麼請注意慎用下面的方法，

這些方法將可能改變單個控件實例的語言列表：

addLanguage(LanguageInfo); // 如果語言列表中沒有該語言則添加

addItem(...); // 如果語言列表中沒有該語言則添加

setItemData(LanguageInfo, Object); // 如果語言列表中沒有該語言則添加

removeItem(...);

removeLanguage(LanguageInfo);

示例

       DemoMultiLangBox.java

    示例2：

public class TestBizMultiLangBox extends TestingFrame {

 

    public TestBizMultiLangBox() {

       // 初始化語言列表

       Locale[] locs = new Locale[] { Locale.CHINA, Locale.US, Locale.JAPAN };

       KDBizMultiLangBox.createLanguageList(locs);

 

       // 創建多語言控件

       KDMultiLangBox mlbA = new KDBizMultiLangBox(); // 創建單行多語言控件

       KDMultiLangArea mlbB = new KDBizMultiLangArea(); // 創建多行多語言控件(形式一)

       KDMultiLangBox mlbC = new KDBizMultiLangBox(KDMultiLangBox.MULTILINE);

       // 創建多行多語言控件(形式二)

 

       // 添加控件到面板中

       cp.add(mlbA);

       cp.add(mlbB);

       cp.add(mlbC);

    }

 

    public static void main(String[] args) {

       (new TestBizMultiLangBox()).setVisible(true);

    }

}
