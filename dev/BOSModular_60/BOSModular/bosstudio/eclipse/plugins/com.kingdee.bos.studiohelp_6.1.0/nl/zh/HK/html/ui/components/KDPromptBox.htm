使用指南

       F7控件是一個特殊的編輯控件，特點是按下F7鍵能夠彈出輔助選框。下面是該控件的使用方法。

       典型流程：

       1．用戶按下F7鍵；

       2．控件彈出輔助選框；

       3．用戶選擇某條目，並點擊確定或取消按鈕，輔助選框關閉；

4．控件根據輔助選框isCanceled方法得到輔助選框狀態；如果是確定狀態（isCanceled返回false）則通過輔助選框的getData方法獲得用戶選擇的數據。注意：數據是從輔助選框到控件單向流動的。

5．控件發出數據「將改變」事件(KDPreChangeEvent)，PreChangeListener對事件參數中的數據進行處理(包括驗證、屬性設置等)，如果preChange方法返回true；控件將data的text屬性設置到控件的編輯框中。

6．控件發出數據改變事件。

 

       屬性的使用：

· Data屬性：控件的數據屬性，Object對象。正常情況下編輯框中顯示的是Data的toString()返回值，在F7業務控件中，編輯框將根據客戶指定格式顯示。

· Selector屬性：彈出式模式窗口，實現KDPromptSelector接口的窗口。用戶通過該窗口選擇數據。

· Dialog屬性：廢棄，被Selector屬性替代。

· Editable屬性：可編輯屬性。如果為true，用戶可以在編輯框中輸入文本。

· Value屬性和Display屬性由KDTable使用，由於和KDTable分佈在不同包中，故可見性為public，不建議客戶程序使用。

 

功能說明：

1．清除功能

按下ctrl鍵，當前具有焦點的F7控件的按鈕功能變為「清除數據」，同時切換圖標。在該狀態下點擊該控件的按鈕將清除數據，並發出ChangedEvent事件。

2．可編輯功能

接口為：

public void setEditable(boolean aFlag);

public boolean isEditable();

在可編輯狀態下，Enter鍵為輸入確認。確認後，控件將編輯框中的文本通過commitEdit方法轉換為一個對象。

3．焦點控制

控件失焦點時如果文字已被手動編輯則執行提交（觸發KDPreChangeEvent事件）。

失焦點有以下幾種情況：

·Tab切換或鼠標激活的切換：如果用戶被手動編輯過則提交數據。

·Enter切換：如果數據已經被提交，則不提交。

·彈出PromptSelector：這種情況當作焦點在本控件處理，不提交數據。

 

事件響應：

1．數據「將改變」事件，發生在臨數據改變之時：

public void addPreChangeListener(PreChangeListener listener);

public void removePreChangeListener(PreChangeListener listener);

客戶程序實現的監聽器需要實現PreChangeListener接口：

public interface PreChangeListener extends EventListener{

    void preChange(PreChangeEvent e);

}

如果在preChange()實現中指定驗證正確e.setResult(PreChangeEvent.S_OK)新數據將被設置到控件Data屬性中，如果驗證正確但是數據不需要更新到控件中使用e.setResult(PreChangeEvent.S_FALSE)，如：在該事件中客戶程序對新舊數據進行比較。如果二者相等，此時希望控件不去更新數據以免不必要的執行過程。

驗證失敗使用e.setResult(PreChangeEvent.E_FAIL)。

PreChangeEvent中包含新\舊數據，通過PreChangeEvent.getData()、getOldData方法獲得。客戶程序除了可以改變新值屬性以外，可以通過setData()方法設置一個全新的新值對象並返回給控件。

 

 

2．數據改變事件，發生在數據改變後：

public void addChangeListener(ChangeListener listener);

public void removeChangeListener(ChangeListener listener);

 

3．選擇框「將彈出」事件，發生在選擇框彈出之前：

public void addSelectorListener(SelectorListener listener);

在彈出選擇框之前，監聽器可以修改控件的某些屬性，特別是和選擇框有關的屬性。參見F7業務控件說明部分。

 

4. Commit事件，發生在用戶在編輯框中按Enter鍵後：

public void addCommitListener(CommitListener listener);

在解析編輯框中的字符串之前發出該事件，事件中帶有編輯框中的文本，使用CommitEvent.getText()方法獲得。其他參見F7業務控件說明部分。

 

常見問題

       Q: 設置給KDPromptBox的data對象的引用可不可以用同一個？比如下面代碼：

Object data = new User();

f7.setData(data);

data = new Admin();

f7.setData(data);

A: 當F7的Data屬性所對應對象的屬性改變時，文本展現也同步更新功能雖已經添加，但是由於該功能提出時所對應的應用場景比較少見，考察可能有類似用法的其它控件比如JComboBox是不支持這種應用（比如改變ComboBox的SelectedItem所對應對象的屬性時，文本展現不會更新），因為通常的設計算法是僅當新舊值不同控件才會更新。 如果是一般性的使用而不追求技巧的話，建議客戶程序使用不同引用對同一屬性進行設置如下：

Object data = new User();

f7.setData(data);

data = new Admin();

f7.setData(data);
