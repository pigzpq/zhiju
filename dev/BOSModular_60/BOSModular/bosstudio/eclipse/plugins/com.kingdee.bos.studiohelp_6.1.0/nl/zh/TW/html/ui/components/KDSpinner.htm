功能列表

基本

上下箭頭為一列和編輯框組成左右佈局

上下箭頭按鈕寬度不變，高度隨著編輯框高度變化。在窗體過小的情況下，允許按鈕本身顯示不全

控件中的編輯框和按鈕顯示效果和基本控件一致

　

Tab鍵切換

切換控件(非控件中的子元素)，焦點只屬於整個控件，控件中的元素沒有焦點狀態

鍵盤方向鍵操作

焦點狀態下，支持鍵盤方向鍵操作，遞增/減編輯框中的數值；按下不放將持續增減。

Enter操作

焦點狀態下， 按回車鍵確定數值，且默認下一個控件獲得焦點

編輯器可設置

允許定制不同的編輯器，遞增/減方法由編輯器提供

步長可設

適用於NumberSpinnerModel，其他model需要定制接口

閾值處理

當到達最大值或最小值，再次改變數值導致超出取值範圍時，自動變為最大或最小值, 該功能由相應的model完成

樣式

使用指南

上下選框能夠是一個能通過上下箭頭進行數據遞增/減選擇的控件，除了具備JSpinner的功能外，它增強了下面的功能：

 

1．步進設置

該功能的適用於spinner的editor的model從SpinnerNumberModel繼承或實現了Stepable接口的情況。前者參見jdk1.4；後者通常是客戶程序設置自定義的editor，該editor擁有一個實現Stepable接口的model。

 

2．嵌入外部控件

出於特殊需要，客戶程序可能需要嵌入外部控件（對spinner配置新的編輯器或者model），該外部控件可能是一個已有的控件，也可能是editor和model分離尚未組合的程序。為了能夠被spinner使用，該外部控件實現時需要注意：

·該控件model實現SpinnerModel接口。

·如果編輯器需要響應enter鍵並做焦點切換，自定editor需要自己實現該功能，因為控件不能「越權」定義客戶編輯器的行為。

·如果editor本身沒有和model關聯，在構造editor時需要添加數據改變監聽器以響應數據變化來改變顯示文本。

·如果需要支持可調步長，model需要實現Stepable接口。

·如果插入到spinner中的editor是組合控件，建議覆蓋hasFocus方法，以使控件正確返回focus狀態。
