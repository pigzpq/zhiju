使用指南

       F7業務控件是在F7通用控件基礎上針對業務需求實現的一個控件。它除了擁有通用控件的特性外，還支持業務數據的綁定。

       業務數據綁定相關的屬性的使用：

1）  displayFormat：顯示期格式，String類型，以"$fieldname$"表示填入字段，"$$"表示"$"符.取值示例："$id$+$name$"。在不可編輯時，焦點和非焦點狀態的數據展現都使用displayFormat，忽略eidtFormat。

2）  editFormat：編輯期格式，其它同displayFormat。

3）  commitFormat：數據提交格式屬性。它是展現格式（上面的displayFormat或editFormat都屬於展現格式之一）的序列，用「;」隔開。格式為：<展現格式1>[;<展現格式2>[...]]。如：」$id$;$name$「 ，又如：」$id$+$name$;$id$+$age$」。以後者為例，在提交數據時生成過濾條件為：(id=001 and name="username") or (id=001 and age=35)。commitFormat暫不支持非String類型字段。如果設置了非String類型的字段作為匹配字段，將查詢不到結果。

4）  queryInfo：查詢對象的PK值，String類型。注意：F7用到的queryInfo中的主對像必須有主鍵字段。

5）  entityViewInfo：查詢條件，EntityViewInfo類型。

6）  多選屬性(enabledMultiSelection), true為支持多選。多選時data為數組, 默認展現數組第一項(下標為0)。多選到單選的切換會將數據[0]作為Data屬性，其它數據丟棄。Data數據類型保持單個元素的類型。

7）  defaultF7UIName屬性：設置F7彈出框的類名, 如果Selector為null, 那麼將使用該類名進行反射創建一個彈出框。如果該類名為null, 那麼將彈出KDCommonPromptDialog。

8）  userData屬性：客戶程序用於存儲業務數據的一個屬性，比如存儲HashMap。

 

如果客戶程序需要擴展功能，以下屬性可以幫助你很容易的自定義控件的行為。下面是控件結構示意圖：

圖中有以下三個可替換構件：

1）  displayFormatter：顯示期格式器，將控件中的data對像格式化為顯示期展現在編輯框中的字串。

2）  editFormatter：編輯期格式器，將控件中的data對像格式化為編輯期展現在編輯框中的字串。

3）  commitParser：對像解析器，將編輯框中的字符串解析為保存在控件中的數據對象。默認解析器是DefaultParser，它的流程如上圖所示。

 

事件：

1．選擇框「將彈出」事件，發生在選擇框彈出之前：

public void addSelectorListener(SelectorListener listener);

在彈出選擇框之前，監聽器可以修改控件的某些屬性，特別是和選擇框有關的屬性。默認情況下的彈出框是KDCommonPromptDialog，在本事件中就可以對彈出框的queryInfo，entityViewInfo屬性進行設置。監聽器還可以在這時使用KDPromptBox.setSelector()設置一個新的彈出框，甚至可以不讓控件彈出選擇框，使用SelectorEvent.setCanceled(true)即可。

 

2. Commit事件，發生在用戶在編輯框中按Enter鍵後：

public void addCommitListener(CommitListener listener);

在解析編輯框中的字符串之前發出該事件，事件中帶有編輯框中的文本，使用CommitEvent.getText()方法獲得。客戶程序可以在事件處理中設置Parser及其屬性。也可以通過CommitEvent.setCanceled(true)取消該次解析的執行。

典型案例分析 - 如何使用自定義的對話框

     1. 確定彈出框的繼承關係

自定義彈出框如果用到了Query對象，並和KDCommonPromptDialog應用場景相似， 可以直接從KDCommonPromptDialog繼承，並遵從其規範（屬性設置和返回值結構）。如果完全未使用到Query對像甚至和業務對像無關，則需要從KDPromptSelector繼承。

    

     2. 定制控件編輯框中的數據展現

如果彈出框從KDCommonPromptDialog繼承，返回值是一個數組。單選時只有第0項有效。如果彈出框從KDPromptSelector繼承，返回值應當是直接所需的data屬性值。返回值通過默認格式化器按照顯示期和編輯期格式轉化為字符串顯示在控件的編輯框中。只有當默認格式化器不滿足客戶程序需要時才需要自己編寫IFormatter實例，並且使用KDBizPromptBox.setXxxFormatter()方法設置到控件中。實現IFormatter接口時，其中方法valueToString(Object o)的參數o就是KDPromptSelector.getData()返回的對象。

    

     3. 如何提交數據

這裡提交的意思是將編輯框中的字符串變為對像存到控件data屬性中。

數據提交通過IParser實現，見控件的commitParser屬性。

同樣KDBizPromptBox提供了默認的數據提交的實現：DefaultParser。該類的功能是將編輯框中的文本按commitFormat進行解析生成條件，然後通過查詢數據庫得到結果集，並構建業務對像設置到控件中。如果該解析器不滿足需要則需要客戶程序定制數據提交的解析器。

 

4．注意事項

自定義了格式化器和解析器後，控件屬性displayFormat、editFormat、commitFormat將轉變為自定義的格式化器或解析器的pattern（applyPattern方法的參數）。

 

示例

       1．直接使用業務控件：

KDBizPromptBox pb = new KDBizPromptBox();

pb.setQueryInfo("com.kingdee.bos.ctrl.extendcontrols.User");

pb.setEditFormat("$id$");

pb.setDisplayFormat("$id$+$name$");

pb.setCommitFormat("$id$+$name$;$id$+$age$");

EASv4.1 新增內容：

一 、F7支持業務對象的維護

1.    需求：從F7選擇對話框可以調出業務對象的序時簿，從而進行各種新增維護刪除操作，關閉序時簿返回F7對話框後自動刷新以體現最新的變化。

2.    設計實現：

A）  Query對像上添加isF7Maintain布爾型擴展屬性，指示該Query是否支持業務對像維護。

B）  Query對像或者Query對象的主實體對像上添加擴展屬性 「listUI」，指明該Query對像相關的業務對像需要使用哪個序時簿進行維護操作。如圖：
 
 

 

 

 

 

 

 

 

 

 

 

 

 

 


  

 

 

 

 

 

 

 

C）  F7選擇對話框中添加「維護」 按鈕，Action 事件響應中首先讀取Query對像上isF7Maintain擴展屬性,如果為true，繼續讀取「listUI」屬性， 如果Query對像上沒有這個擴展屬性，則從Query對象的主實體對像上去獲取。註：如果isF7Maintain為false，或者Query對像和Entity上都沒有 ListUI 擴展屬性的值，維護按鈕將不可見。

 

    

 

   

二 、 F7支持通用過濾

1. 術語解釋：

1.1 原始條件：用戶在點開F7之前，程序設置的過濾條件。

1.2 自定義條件：用戶在F7選擇對話框中彈出用通用過濾界面，在通用過濾界面上定義的過濾條件。

 

2.       需求：

2.1   F7選擇對話框彈出時，根據原始條件取數。

2.2   用戶在F7選擇對話框中可以調用通過過濾對話框設置自定義過濾條件，返回後F7根據原始條件和自定義過濾條件的交集取數。

2.3   當沒有自定義過濾條件時，按「刷新」按鈕按原始條件取數，用戶設置過自定義過濾條件後根據原始條件和自定義過濾條件的交集取數。

2.4   用戶自定義過濾條件方案不保存，但一次調用的過程中可以緩存。

   

三、本版F7修復的bug

 

1 一個F7控件來回切換邦定不同的Query時，引發QueryInfo為空，拋異常。

     原因：程序員切換前，會 setSelector（null）， 這樣下次邦定QueryInfo時 會在CreateSelector()中新實例化一個Selector，並且刷新標誌沒有置為true，這樣在ConfigSelector（）中不會為 Selector set QueryInfo。造成 進入 Selector中 fillTable時 報出 nullPointException。

     解決辦法： 在CreateSelector（）中新實例化Selector時把refresh標誌置為true。

3.       F7顯示日期字段時把日期、時間一起顯示

在 fomatSingleObject() 中 對 字段類型進行判斷，如果是 java.util.Date 及其子類，截取日期部分，顯示。
