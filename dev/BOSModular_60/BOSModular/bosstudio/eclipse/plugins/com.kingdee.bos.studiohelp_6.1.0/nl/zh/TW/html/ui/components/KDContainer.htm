使用指南

       可展容器是一個特殊的容器，特點是可伸縮。下面是該控件的使用方法。

       屬性的使用：

·State屬性：容器的狀態屬性，有展開和收縮兩種狀態：

        1）SHRINK 

    2）SPREAD

·EnableActive：設置容器是否可以伸縮活動。值為false時不能伸展或收縮。

·Title屬性：標題屬性，為文本。

·TitleLength：標題長度。如果不滿意默認的標題長度，可以自定義長度。默認值為TITLELABEL_MAXLEN。

·TitleStyle：設置標題的風格。有兩種：可展按鈕風格（ACTIVESTYLE）、主界面風格（STATICSTYLE）、標籤風格（LABELSTYLE）。如下圖：

主界面風格：

有一個幾何面板作為裝飾

可展按鈕風格：

按鈕和標籤組成，按鈕是箭頭按鈕

標籤風格:

純文本標籤，比主界面風格少了幾何裝飾面板，同樣當EnableActive為false時，圖標變為箭頭表示可展。

 

·Gap屬性：設置最右邊的命令按鈕（標題條右側）與容器右邊框的距離。標籤風格風格默認距離為0像素，其他風格默認為8像素。setGap(-1)表示使用默認距離。

 

方法的使用：

·add(Action)：在標題條上添加一個按鈕，用法同在JToolBar上添加一個按鈕。

·removeButton(JButton): 移除標題條上的一個按鈕，參數是add(Action)方法中返回的JButton對象。

·removeAllButton: 移除標題條上所有的按鈕。

 

使用KDContainer時建議使用KDContainer.add(Action)方法添加標題條按鈕，使用方法和ToolBar的add(Action)一樣。示範如下：

 Action deleteAction = new DeleteAction("Delete", new TestIcon("D"));
 kdContainer.add(deleteAction);

...

 class DeleteAction extends AbstractAction {
  public DeleteAction(String name, Icon icon) {
   super(name, icon);
  }
  public void actionPerformed(ActionEvent e) {
   // TODO Auto-generated method stub
  }
 }

而addButton是為添加一些自定義的按鈕準備的接口。

 

在可展容器中放置控件：

為了在容器上放置控件，需要使用getContentPane()獲得內容面板，添加控件和設置佈局管理器。

 

事件響應：

可展容器發出活動容器事件：

public void addActiveContainerListener(ActiveContainerListener);

public void removeActiveContainerListener(ActiveContainerListener);

其中ActiveContainerListener用於處理容器事件，包括以下事件類型：

·SPREAD展開命令

·SHRINK 收縮命令

在容器形狀發生變化時，客戶程序需要刷新佈局；當最大化事件發生時，客戶程序需要根據當前父容器的大小和排布方式重新設置該容器控件的大小和其它子控件的大小，並最後刷新父容器佈局。
